What's new:

  • You can now open multiple instances of the Test Case Browser to make navigation in larger projects more easy.
  • The workspace dialog now lets you set a workspace as the default.
  • In the "File" menu, you can now restart the application and also switch workspaces.
  • You can configure a default database that you connect to when starting the tool.
  • You can also specify a default project that is automatically loaded when connecting to the database.
  • The testexec and dbtool applications can now accept the -dburl parameter instead of the workspace parameter.
  • We have added support for a Chronon Recording Server to the monitoring agents for an AUT.
  • The version of Chronon used for integrated recording has been updated to version 3.

The list above is just an overview. For a more complete guide to new features, read the release notes.
