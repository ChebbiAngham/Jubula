3.3.3 Migrating to newer versions of the Database

If a new version of Jubula uses a new Database scheme, you will be informed of this when attempting to connect to the Database .

From the information dialog, you can start the migration wizard. This wizard removes the old Database scheme (and any Projects in it) and automatically creates the new scheme for the current version.

You can choose which Projects (exported from the old version) you want to import into the new Database .

If you do not have sufficient privileges to clear your Database , then you must manually clear it using a Database administration tool.
