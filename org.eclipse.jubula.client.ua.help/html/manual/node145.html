3.12.2.1 Testing independently started AUT's

To be able to test multiple AUT's that are not started by each other, the following criteria must be met:

  • The AUT's are either written with the same toolkit (e.g. Swing) or,
  • you have specified your Project at the concrete level, and will only be testing areas of the AUT's that can be tested with the actions that are valid for all AUT types (i.e. no RCP-specific components are involved in the test) 7.
  • The AUT's are all defined in the same Project .
  • The first AUT can either be started using the autrun command 3.5.4 or via an AUT configuration 3.5.3. Any other AUT's required for the Test Job must have been started with the autrun command.
To run Test Jobs from the test executor, all AUT's for the test run must be started with the autrun command.
