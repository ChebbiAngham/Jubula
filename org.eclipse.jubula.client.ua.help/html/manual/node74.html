3.5.3 Configuring AUT's to be started via Jubula

Once you have created a Project 3.4.1 and defined an AUT 3.5.2, you can add and edit AUT configurations. Use this option if you want Jubula to start your AUT for you. The details in the AUT configuration tell Jubula how to start the AUT , e.g. on which machine. An AUT can have multiple configurations (for example, for local and remote testing).

If you want to start your AUT yourself, and have Jubula connect to it, then use the autrun command to start the AUT 3.5.4. In this case, you do not need to create an AUT configuration.

The AUT configuration dialog has three different levels of detail: basic, advanced and expert.

See the sections below for information on the different levels.
