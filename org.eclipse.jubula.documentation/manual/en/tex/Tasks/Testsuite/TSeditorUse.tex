\gdsuites{} are used to organize and prepare \gdcases{} to be executed. 

To create a \gdsuite{}, the \gdtestsuitebrowser{} must be visible.
 If it is not visible, change to the \specpersp{} and select:\\
\bxmenu{Window}{Show View}{Test Suite Browser}.

\begin{enumerate}
\item Select:\\
\bxmenu{New}{New \gdsuite{}}{}\\
from the context-sensitive menu in the \gdtestsuitebrowser{}. 

 \item Name  the new \gdsuite{} when prompted with a meaningful name. It is a good idea to have naming conventions \bxpref{BPTestSuiteNames} for \gdsuites{}. 
\item Click \bxcaption{OK}. 
\item The \gdsuite{} you created will appear in the \gdtestsuitebrowser{} under the category \bxname{\gdsuites{}}. 
\item Once you have created a \gdsuite{}, you can:
\begin{itemize}
\item Add \gdcases{} to it to be executed \bxpref{TasksEditorAdd}.
\item Configure the \gdsuite{} in the \gdpropview{} \bxpref{confsuite}.
\item Rename the \gdsuite{} \bxpref{TasksBrowserRename}.
\item Delete \gdcases{} from it \bxpref{TasksEditorDelete}.
\item Add the \gdsuite{} to a \gdjob{} to be executed along with other \gdsuites{} \bxpref{WorkingWithJobs}.
\end{itemize}
\bxtipp{You can filter in the \gdtestsuitebrowser{} using the field at the top. Use star \bxshell{*} as a wild card.}
\end{enumerate}

\bxtipp{You can combine multiple \gdsuites{} to create a \gdjob{} \bxpref{WorkingWithJobs}. This is useful for testing multiple \gdauts{} (or instances of the same \gdaut{} in one test.}
