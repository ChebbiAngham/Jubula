If you want to test a \bxname{Rich Client Platform} application with \jb{}, you
must first unzip our \bxcaption{RCPAccessor} plugin into your
\gdaut{}. This can be done as follows:

\begin{enumerate}
\item Locate your \jb{} installation directory.
\item Extract the content of the \bxname{rcp-support.zip} folder into the \bxname{plugins} directory for your RCP \gdaut{}.
\bxtipp{When you install a new version of \jb{}, you must repeat these steps with the new RCP accessor. We recommend starting your \gdaut{} once with \bxshell{-clean} to ensure that the new accessor is used. }
\item RCP applications generally have a configuration/config.ini file which contains the parameter \bxname{osgi.bundles}. This parameter may need to be modified to allow the RCP Accessor plugin to load on \gdaut{} startup. The \bxname{org.eclipse.update.configurator} plugin automatically loads all plugins found in the plugins directory, which means that the RCP Accessor should start with the \gdaut{} if \bxname{org.eclipse.update.configurator@3:start} is already defined in the \bxname{osgi.bundles} parameter. Otherwise, you may need to add  \bxname{com.bredexsw.guidancer.rcpaccessor} to the end of the \bxname{osgi.bundles} parameter.
\end{enumerate}

If you do not follow the above steps, the \gdserver{} will not be able to
communicate with your \gdaut{}!
