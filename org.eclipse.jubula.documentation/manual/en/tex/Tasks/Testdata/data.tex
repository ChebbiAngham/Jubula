\index{Parameter}
\index{Test data}
 
Most actions supported by \gd{} require one or more \bxname{parameters} (e.g. how often to click, what text to enter). 
Once you have added \gdcases{} or \gdsteps{} to a \gdcase{}, you will see which parameters are required by this action in the \gdpropview{}. 

\bxtipp{You can also enter \bxname{data sets} in the \gddatasetsview{} \bxpref{TasksDataSetsView} and you can create central data sets to be reused throughout the \gdproject{} \bxpref{TasksCentralData}. }

\gd{} lets you be very flexible with data, so you have a choice about what to enter in the \gdpropview{}:

\begin{enumerate}
\item Concrete values \bxpref{TasksTestdataConcrete}.
\item References \bxpref{TasksTestdataReferences}.
\item Variables \bxpref{TasksVariables}.
\item A mixture of the above \bxpref{TasksConcatenation}.
\item An Excel file \bxpref{TasksExcel}.
\item A reference to a central data set \bxpref{TasksCentralData}.
\end{enumerate}
