You can write tests for iOS \gdauts{} when you select the toolkit \bxname{concrete}, \bxname{mobile} or \bxname{iOS} in the \gdproject{} properties.

\bxtipp{Choose \bxname{mobile} if, at a future point, you will want to use this \gdproject{} for cross-platform mobile testing.}

When you select \bxname{mobile} as the \gdproject{} toolkit, the library \gdprojects{} \bxname{unbound\_modules\_concrete} and \bxname{unbound\_modules\_mobile} are automatically reused in your \gdproject{}. 

When you select \bxname{iOS} as the \gdproject{} toolkit, the library \gdprojects{} \bxname{unbound\_modules\_concrete},  \bxname{unbound\_modules\_mobile} and \bxname{unbound\_modules\_iOS} are automatically reused in your \gdproject{}. 


The actions in the concrete library are described in the reference manual \bxextref{\gdrefman}{ref,actparam}.
